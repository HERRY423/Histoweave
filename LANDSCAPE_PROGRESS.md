## 试点实验结果关键发现

**扩展后（7 方法 × 6 数据集）：**

| 方法 | 胜场 | 均值 ARI | 生态位 |
|------|:--:|------|------|
| **gaussian_mixture** | 6/6 | 0.999 | 全覆盖 — 在 Voronoi blob 数据上最优 |
| **kmeans** | 0 (平 4) | 0.999 | 全覆盖 — 与 GMM 并列但从未独赢 |
| **spectral** | 0 | 0.985 | 5/6 接近最优 — 多域场景强 |
| **agglomerative** | 0 | 0.903 | 1/6 — 安全备选 |
| **optics** | 0 | 0.007 | 0 — Voronoi 连通域不适用 |
| **dbscan** | 0 | 0.000 | 0 — 同 optics |
| **mean_shift** | 0 | 0.000 | 0 — 同 dbscan |

**核心发现：**
1. 在 Voronoi blob 数据上，基于质心的方法（GMM, kmeans）天然占优——这**证实了方法性能与数据特征之间存在可预测的关系**
2. 密度方法（DBSCAN, OPTICS, MeanShift）全败——它们在 Voronoi 连通域上无法工作，但在**真实不规则组织**中可能是最佳选择
3. 这恰恰验证了核心假说：**不存在万能方法，必须在数据特征空间中为每个方法找到其生态位**

**推荐引擎验证：**
- 对新合成数据集正确推荐了 gaussian_mixture + kmeans 共识集成
- 识别出 dense_regular 为最相似参考数据集（cosine sim=0.40）
- 集成策略："共识投票 — 一致区域直接赋值，分歧区域标记为不确定"
