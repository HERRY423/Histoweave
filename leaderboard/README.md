# HistoWeave leaderboard

Static single-page site that renders HistoWeave's domain-detection ARI
benchmark as a filterable table, heatmap, and per-method small-multiples.

Deployed automatically to GitHub Pages on every push to `main` via
[`.github/workflows/pages.yml`](../.github/workflows/pages.yml).

## Files

| File          | What it does |
| ---           | --- |
| `index.html`  | Layout, sections, and control elements. |
| `styles.css`  | Phylo-palette-based styles (accessible ink on paper background). |
| `main.js`     | Loads `data.json`, renders the table + heatmap + small-multiples, wires filters and sort. Uses D3 v7 via jsdelivr. |
| `generate.py` | Rebuilds `data.json` from the benchmark CSV artefacts. Reads `5x15_spatial_aware/`, `5x10_dlpfc_benchmark/`, and `benchmark_crossplatform/` when present; missing files are skipped, not fatal. |
| `data.json`   | Machine-readable ARI + timing feed. Regenerated by CI; you can commit it or leave it uncommitted (the workflow rebuilds it either way). |
| `README.md`   | This file. |

## Local preview

```bash
python leaderboard/generate.py     # rebuilds data.json
python -m http.server -d leaderboard 8080
# open http://localhost:8080
```

The page fetches `data.json` from the same directory as `index.html`, so
serving via `http.server` (or any static host) works out of the box.

## Data feed schema

`data.json` is deliberately small and stable:

```json
{
  "protocol": "histoweave.leaderboard.v1",
  "generated_at": "2026-07-14T15:00:00Z",
  "datasets": [
    {"id": "151673", "platform": "Visium", "tissue": "DLPFC", "n_obs": 3611, "n_domains": 7}
  ],
  "methods": [
    {"name": "banksy_py", "family": "spatial_aware", "summary": "..."}
  ],
  "records": [
    {"dataset": "151673", "method": "banksy_py", "seed": 42, "ari": 0.273, "seconds": 1.9, "family": "spatial_aware"}
  ]
}
```

`family` is either `sklearn` or `spatial_aware`. The client-side JS derives
mean ARI, best ARI, and win counts on the fly, so adding a new method
requires no code changes — regenerate `data.json` and reload.

## Design notes

- **No build step.** GitHub Pages serves static files, so we ship
  `index.html`/`main.js`/`styles.css` verbatim. D3 loads from jsdelivr; if
  the repo enforces a no-third-party-CDN policy later, drop a vendored
  `d3.min.js` into this folder and update the `<script src>` in
  `index.html`.
- **Palette.** Phylo colours only (`#000000, #ECE9E2, #FAF9F3, #E9ED4C,
  #FF9400, #75A025, #FD9BED, #0279EE`). Accent-orange marks the winning
  method per dataset; accent-green flags spatial-aware methods.
- **Reproducibility.** The workflow reruns `generate.py` on every push, so
  the site always matches the CSV artefacts committed to the branch. If
  the CSVs move, update the paths in `generate.py`.
- **Accessibility.** All colour-coded cells also carry the numeric ARI in
  text, tooltip titles cover cell provenance, and the table uses semantic
  `<th>`/`<td>` with row-level headers.
- **Progressive enhancement.** With JS disabled the header, hero, and
  section headings still load; only the interactive table/heatmap/panels
  require D3.
