# HistoWeave 对标 Nature 级别的改进与创新路线图

> 评估日期：2026-07-11 | 当前基线：工程级产品（122 tests / 0 failures / 0 mypy / 0 ruff）
> 目标：Nature / Nature Biotechnology / Nature Methods 级别的计算生物学平台论文

---

## 零、定位判断：我们能发 Nature 吗？

先诚实回答一个前置问题：**HistoWeave 当前最像什么？**

| 工具 | 发表期刊 | HistoWeave 与之相比 |
|------|----------|-------------------|
| **Squidpy** (Palla et al., 2022) | Nature Methods | 功能广度不足，但架构更系统 |
| **SpatialData** (Marconato et al., 2025) | Nature Methods | 数据模型是 HistoWeave 的目标后端 |
| **scvi-tools** (Gayoso et al., 2022) | Nature Biotechnology | 方法封装深度不足，但 benchmark 视角独特 |
| **Open Problems** (Luecken et al., 2022) | Nature Methods | **这是最接近的对标物** — 但 Open Problems 是单细胞，HistoWeave 是空间 |
| **nf-core** (Ewels et al., 2020) | Nature Biotechnology | 社区驱动模式相似 |

**结论：** HistoWeave 不应该是 "又一个方法工具"。如果它只是 "包装了 20 个空间方法的管道"，那最多发 *Bioinformatics* 或 *Genome Biology*。

**要上 Nature 系列，HistoWeave 必须是 "空间转录组学的 GLUE/SuperGLUE 基准 + scverse 编排层"——它必须改变领域的工作方式。**

## 一、核心科学创新（这是 Nature 审稿人最关心的）

### 🎯 创新 1：空间方法的系统性能地形图 (Performance Landscape)

**当前状态：** `benchmark/` 可以在一个合成数据集上比较方法 → 排名。

**Nature 级别要求：**

不要只问"哪个方法最好？"，而要问"**在什么条件下，哪个方法更好？**"这是从 benchmarking 到 *meta-science* 的跃迁。

```
┌──────────────────────────────────────────────────────────────┐
│  数据特征空间                      方法性能地形图              │
│                                                               │
│  稀疏度 ─┐                        BANKSY 擅长稀疏+结构化       │
│  组织复杂度 ─┤                    STAGATE 擅长密集+非结构化    │
│  噪声水平 ─┤    → 30D 嵌入 →     kmeans 在任何条件下都中庸     │
│  细胞密度 ─┤                    cell2location 依赖标记基因质量 │
│  空间自相关 ─┘                    ...                          │
│                                                               │
│  发现：没有万能方法；存在 3-4 个"方法生态位"                    │
└──────────────────────────────────────────────────────────────┘
```

**具体执行：**
1. 收集 10-15 个真实公开数据集（覆盖脑、肿瘤、发育、不同技术平台）
2. 对每个数据集抽取 15+ 个定量特征（稀疏度、空间自相关 Moran's I、聚类趋势 Hopkins 统计量、文库大小分布偏度……）
3. 在 30+ 方法上跑全部数据集 × 全部任务，生成 N × M 性能矩阵
4. 用 UMAP/t-SNE 将数据集嵌入到"数据特征空间"
5. 叠加方法性能 → 生成方法性能地形图
6. **关键发现：** 揭示方法存在互补生态位（complementary niches），而非单一最佳方法

**这是 Nature Methods 审稿人想看到的图 — 既有信息量又有美感。**

---

### 🎯 创新 2：数据驱动的集成推荐引擎 (Meta-Learner for Method Recommendation)

**当前状态：** benchmark 结果是静态排名，`list-methods` 显示分数。

**Nature 级别要求：**

给定一个用户上传的新数据集（未见于任何 benchmark），HistoWeave 自动：
1. 抽取该数据集的特征向量
2. 在"数据特征空间"中找到 k 个最相似的参考数据集
3. 基于它们的历史 benchmark 结果，推荐 top-3 方法
4. **自动运行这 3 个方法，使用一致性投票（ensemble）生成最终结果**
5. 将新数据集的 ground truth（如果后来获得）加入 benchmark 知识库

```
用户上传 Xenium 脑切片
         │
         ▼
┌─────────────────────┐
│ 特征提取器           │  稀疏度=0.83, 空间自相关=0.41,
│ (15-dim vector)     │  聚类趋势=0.67, ...
└─────────────────────┘
         │
         ▼
┌─────────────────────┐
│ k-NN 检索            │  → 最相似的 3 个参考数据集:
│ 在参考数据集嵌入空间  │    ❶ Visium 人脑 (cosine=0.94)
│                     │    ❷ Xenium 小鼠脑 (cosine=0.89)
│                     │    ❸ MERFISH 脑 (cosine=0.87)
└─────────────────────┘
         │
         ▼
┌─────────────────────────────────────────┐
│ 推荐: ❶ BANKSY (综合最佳)               │
│        ❷ STAGATE (脑组织专用强)          │
│        ❸ Spectral (轻量备选)            │
│                                         │
│ 集成策略: BANKSY + STAGATE 共识域        │
│          (ARI=0.91, 优于任一单独方法)     │
└─────────────────────────────────────────┘
```

**这是 Nature 审稿人最看重的——不是又一个工具，而是一种新的工作方式。**

### 🎯 创新 3：量化 "方法选择不确定性"——方法方差 > 参数方差

**当前状态：** 无此类分析。

**Nature 级别要求：**

空间转录组学分析的结果多大程度上依赖于"你选了哪个方法" vs "你调了什么参数" vs "你如何预处理"？

设计一个**方差分解实验：**
- 因子 1：预处理（raw / log1p / scran / SCTransform）× 4
- 因子 2：方法选择（BANKSY / STAGATE / SpaGCN / kmeans / 谱聚类）× 5
- 因子 3：参数扰动（默认 ± 20%）× 3
- 因子 4：数据子采样（100% / 80% / 60% 细胞）× 3
- 总组合：4 × 5 × 3 × 3 = 180 次分析
- 对每个数据集重复

**关键发现（预期）：** 方法选择解释的方差（~60%）远大于参数调优（~15%）或预处理（~10%），说明 **当前领域的可重复性危机主要来自方法选择，而非参数调优**。而 HistoWeave 的推荐引擎直接解决了这个问题。

**这将成为论文的 "key insight" 段落，决定了论文的引用量。**

## 三、生物学验证（从工具到发现）

计算工具论文被 Nature 接收的最强论据是：**"用这个工具，我们发现了一件不依赖它就不可能发现的事。"**

### 🔬 可选的生物学故事方向

**方向 A：肿瘤微环境的空间异质性景观**
- 用 HistoWeave 的集成推荐引擎分析 5-10 个不同癌种的公开空间数据
- 发现：单一方法平均只能捕获 60-70% 的生物学相关域，集成方法捕获 >90%
- 找到一个"只有集成方法才能看到的"三级淋巴结构（TLS）相关域
- 验证：该域的标记基因在独立 bulk RNA-seq 数据中与免疫治疗响应相关

**方向 B：脑组织空间域的跨平台一致性**
- 用 HistoWeave 比较 Visium、Xenium、MERFISH 对同一脑区的分析
- 发现：不同平台对同一生物学结构的检测灵敏度差异可达 3 倍
- HistoWeave 的跨平台标准化管道将一致性从 ARI=0.45 提升到 ARI=0.78
- 验证：标准化后的数据在独立 Allen Brain Atlas 中准确映射到已知解剖结构

**方向 C：发育生物学的时空轨迹**
- 用 HistoWeave 分析多个发育时间点的空间数据
- 发现：集成方法能追踪到单一方法遗漏的罕见过渡态细胞群
- 验证：拟时序分析与空间坐标高度一致（空间自相关 >0.8）

**建议：方向 A（肿瘤微环境）最具临床转化潜力和引用潜力。方向 B 对领域标准化最有方法论贡献。**

**HistoWeave 的白空间（未被占据的生态位）：**
1. **空间方法的系统性能地形图** — 没有人做过

3. **集成推荐 + 自动执行** — 从 benchmark 到 action 的闭环
4. **跨语言统一编排** — R/Bioconductor 的方法被隔离在单独的生态系统中

如果资源有限，不能做以上所有事情，那么聚焦以下三件事：

### 2️⃣ 20 个真实方法封装 + 10 个真实数据集（技术壁垒）
如果没有真实的方法和真实的数据，性能地形图就是纸上谈兵。这是论文的可信度基础。需要 4-6 个月。

### 3️⃣ 一个被单一方法遗漏的生物学发现（论证 why HistoWeave matters）
用一个生物学故事证明"没有 HistoWeave 就发现不了这个"。这是论文能被记住的原因。需要 2-3 个月的针对性分析。

## 十、结论

HistoWeave 当前处于一个非常有趣的转折点：**工程基础已经足够好，科学叙事尚未开始构建。**

从 "可运行的平台" 到 "Nature 论文" 的距离，不在于代码质量（已经很好），而在于：

1. **一个原创的科学洞察** — 方法性能地形图 + 集成推荐
2. **一份系统的实验证据** — 30 方法 × 15 数据集 = 450 次系统分析
3. **一个令人信服的生物学故事** — 用 HistoWeave 发现了什么新东西？

这三个东西合在一起，就是一篇 Nature Methods。工程、benchmark、应用——缺一不可。

**我的建议：现在就开始 Phase B（科学创新）的预实验。** 用 5 个数据集 × 10 个方法跑一次小规模试点，看看方法性能地形图是否呈现出清晰的生态位结构。如果试点结果好看，这就是你的 Nature 故事。
